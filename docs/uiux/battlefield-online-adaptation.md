# 線上戰場改編說明（Battlefield Online Adaptation）

> 官方規則語義：`docs/rules/official-start-guide-ui-analysis.md`
> 實作：`src/components/Board.tsx` ＋ `src/ui/game/`＋`src/ui/game/game.css`

## 官方實體場地的規則語義

官方場地墊為**兩人面對面的鏡像佈局**：各自面前由左至右排 Power Charger、
Set Zone A/B/C、Deck、Abyss；Battle Zone 靠中線相對；中央放 Chronos 錶盤與
メダル；HP 以實體計數器另置。它服務的是：實體卡牌的可達性（手能搆到）、
對坐視角（各自讀自己的牌）、以及公開資訊互見。

## 為什麼不直接照搬實體場地

1. **視角不同**：螢幕是單一玩家視角，不需要鏡像可達性；對手區域只需「可讀」不需「可操作」。
2. **空間不同**：實體墊是固定 16:9 桌面；線上要同時服侍 1920×1080 與 360×740，機械照搬會把手機版壓成不可玩的縮圖。
3. **資訊密度**：實體卡牌本身自帶所有資訊（拿起來看）；線上卡圖只有數十 px，規則關鍵數字（攻擊力、總 Power、剩餘牌數）必須被 UI 外置、放大、常駐。
4. **實體儀式無法平移**：互換洗牌、「嫌（やぁ）」口令、手動推 メダル 在線上由引擎與動畫代行。

## 線上版保留的官方語義（不可妥協清單）

- 全部 8 個官方區域及其名稱、用途、可見性規則（見 rule-to-ui-mapping.md）。
- 對手在上、玩家在下的對坐隱喻；Battle Zone 隔著 Chronos 相對。
- Chronos 晝夜配色 = 官方（夜青/晝紅）；起始真夜中；位置/時段常駐可讀。
- 攻擊力差=傷害、PC 不足=0、STP 決定離場去向、抽不到牌敗北 — 全部有對應 UI 呈現。

## 線上版重新組織的部分（與理由）

| 區域 | 設計 | 理由 |
|------|------|------|
| **Chronos** | 戰場正中央、桌面 clamp 至 16rem 的大錶盤；雙弧官方紅藍；全場背景隨晝夜微偏移 | 它是本作規則的主角（晝夜決定攻擊力與效果順序），必須是核心視覺而非角落資訊；背景偏移讓玩家不看錶盤也能感知時段 |
| **Battle Zone** | 兩張卡槽夾著 Chronos 水平並列，槽下常駐「生效攻擊力」讀數（晝夜著色、PW 不足=0+警示） | 官方傷害=攻擊力差；比較必須一眼完成，不能靠 hover。與 Chronos 並列強調「時間決定戰力」 |
| **Set Zone A/B/C** | 玩家/對手各一列；每槽帶用途 hint（設置①/設置②・敗者/持續附魔）；C 區虛線＋青色 | 官方三槽用途不同（勝敗出牌數、Area Enchant 常駐），泛用卡槽會誤導；C 區視覺區隔表達「跨回合持續」 |
| **Power Charger** | 堆疊縮圖＋常駐「總 Power N」大字；點擊開來源摘要 | 官方語義是「資源總量」，總值比逐張卡重要；來源可查（STP 明細）但不佔戰場空間 |
| **Deck / Abyss** | 堆疊＋計數；Deck 不可檢視（資訊隱藏）、Abyss 點擊開摘要 | 剩餘牌數=敗北攸關必須常駐；Abyss 是公開資訊但低頻查閱，收進 sheet |
| **HP** | 雙方 PlayerStatus 常駐 bar（己方 volt / 對手緋紅），分級警示 | 官方 HP 計數器的線上等價物，勝敗攸關不許捲出畫面 |
| **Hand** | 桌面扇形 hover 抬起；觸控橫滑 strip、每張 ≥44px、兩段式 tap | 實體「拿在手上」的替代：桌面追求手感、觸控追求誤觸安全 |
| **Card Detail** | 桌面右欄常駐 / 觸控 bottom sheet（同一 CardDetailBody） | 替代「把卡拿起來看」；單一實作避免三套 popover |
| **Log / Phase / Action** | 頂欄階段軌＋PhaseIndicator 常駐；log 側欄（≥1180px）或 sheet；主行動固定 ActionDock | 實體對局的口頭溝通（該誰、做什麼）由 UI 顯性化 |

## 各終端佈局策略

- **桌面（≥1024）**：完整對坐戰場——對手列（狀態+手牌背+區域列）/ 中央 Chronos+BattleZone / 玩家列（區域列+手牌扇+ActionDock）；≥1180 右側詳情+log 欄；低高度（≤780）卡牌以 vh 錨定壓縮。
- **平板直屏（768–1023）**：同一語義 DOM 直向重排；對手壓縮為狀態列＋可橫滑區域 chips；Chronos 因縱向空間充裕反而放大。
- **手機（<768）**：對手簡版 → 中央戰鬥核心 → 玩家區域列 → 底部手牌 strip＋拇指區主按鈕；深淵/充能/詳情全部走 bottom sheet；橫屏（≤500h）卡牌改 vh 錨定、標籤隱藏，保證不靠縱向滾動可玩。
- 詳細斷點與驗證：`docs/uiux/responsive-layout-system.md`（smoke 16 案例）。

## 舊版偏差與新版修正

見 `official-start-guide-ui-analysis.md` §17：D1 晝色（琥珀→官方紅）、D2 攻擊力比較外置、D3 A/B/C 用途區分、D4 總 Power 常駐 — 均已在本版修正。無需要改動核心邏輯的規則偏差。
