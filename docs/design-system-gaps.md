# Design System Gaps

審查日期：2026-07-03  
基準：`docs/ui-style-guide.md`、`src/App.css` tokens、`src/components/ui/*`。

## Phase 1 Resolution Note

2026-07-04 Phase 1 已補齊 semantic token foundation，並完成 TSX usage migration：

- Resolved for P1: typography/tracking token、popover/sheet/selected/glow/inset shadow token、z-index token、Battle canvas color token、pattern token、Admin/I18n responsive CSS semantic variable usage。
- Still open for Phase 2/3: component abstraction gaps（IconButton、ResponsiveToolbar、DataList、CardDetailSurface）、Dialog/Sheet focus management、Feedback custom tag color policy、legacy `src/App.css` page/game CSS raw value removal。

| 類型               | 缺失內容                                                                                    | 造成問題                                                                   | 建議補充                                                                                       |
| ------------------ | ------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------- | ------------------------------ | --------------------------------------------- |
| Color token        | 舊變數 `--bg-*`, `--panel`, `--line`, `--muted`, `--danger`, `--teal` 還與新 token 並存     | 新舊色彩語義混用；頁面難判斷可用色票                                       | 建立 migration map，標記 deprecated token；新增 lint/rg audit 禁止新用舊 token。               |
| Color token        | Feedback tag 任意 `tg.color`、I18n/Admin CSS 硬編碼 hex、Chronos SVG/canvas hardcoded color | 對比不可控，主題一致性下降                                                 | 增加 `--tag-*` 或 controlled custom swatch policy；SVG/canvas 也需讀 CSS token。               |
| Color token        | Gradient primary/hero 未定義                                                                | Online quick match/Auth/Landing 自行建立 gradient primary                  | 補 `Button emphasis="hero"` 或 `--gradient-primary-action` token，限制用途。                   |
| Typography token   | `Kicker`/`MetaText`/`ControlLabel` 缺失                                                     | 大量 `text-[10px] uppercase tracking-[0.3em]` 散落，手機文字擠壓           | 新增 `Kicker`, `MetaText`, `ControlLabel` components；手機自動降低 tracking。                  |
| Typography token   | 8px/9px 字級使用邊界不清                                                                    | Battle log/badge/Deck metadata 可讀性不足                                  | 規定 `<10px` 只可用於非決策 badge；操作與狀態最小 12px。                                       |
| Spacing token      | Low-height density 未定義                                                                   | 1366x768/1280x720 只能靠頁面各自壓縮                                       | 補 `density="compact                                                                           | default                        | roomy"`，適用 Panel/PageShell/Board/Toolbar。 |
| Spacing token      | Toolbar/control gap token 不足                                                              | Feedback/Deck/Admin toolbar wrap 後密度不一致                              | 補 toolbar spacing scale：`toolbar-gap-sm/md`, `control-gap`, `section-gap`。                  |
| Radius token       | 基本 radius 已有，但互動卡/Sheet/Dialog 的使用規則未完整                                    | 部分 modal/sheet 使用 `rounded-md`，卡片/面板混用自寫 CSS                  | 在 component variants 中固化 radius，不允許頁面直接指定大 radius。                             |
| Shadow token       | `--shadow`/`--shadow-soft` 不足以覆蓋 popover/sheet/glow                                    | `shadow-[0_32px_96px...]`、`shadow-[0_30px_80px...]` 散落                  | 補 `--shadow-popover`, `--shadow-sheet`, `--shadow-selected`；禁止任意 shadow。                |
| Z-index token      | `Sheet` 用 `z-[100]`，GameTutorialOverlay 用 `z-[1000]`，Battle pause 用 `z-50`             | 浮層疊放順序不透明，modal/toast/overlay 可能互蓋                           | 補 `layer` utility 或 CSS classes：dropdown/sticky/overlay/modal/toast/max；全部走 token。     |
| Breakpoint token   | 只有 Tailwind breakpoint，缺少產品語義 tier                                                 | `lg=1024` 對 Battle/Deck/Admin 太早，平板橫屏被當桌面                      | 在 docs/DS 中定義 `desktopSidebar >= 1180`, `lowHeight <= 780`, `touch <= 820`。               |
| Breakpoint token   | Container query rule 缺失                                                                   | 工具頁只能用 viewport width，不能按 workspace 寬度調整                     | 補 container roles：page/workspace/toolbar/card-grid；優先用 container query 管 grid/toolbar。 |
| Component variant  | Button 缺 `icon`, `hero`, `segmented`, `toolbar`, `danger-outline`                          | 頁面自行寫 icon button、hero gradient、tabs                                | 擴充 Button 或新增 IconButton/SegmentedControl，避免 inline class。                            |
| Component variant  | Panel 缺 `interactive`, `status`, `danger`, `compact`                                       | Auth/Admin/Feedback/RoomPanel 自建 panel tone                              | 擴充 Panel tone/density；RoomPanel 只包語義，不重寫外觀。                                      |
| Component variant  | Dialog 缺 `confirm`, `detail`, `fullscreen-mobile`, sticky footer/body slots                | Feedback/Board/AppDrawer 自建 modal/sheet                                  | Dialog/Sheet 增加 slots 與 mobile presentation；ConfirmDialog semantic wrapper。               |
| Component variant  | ResponsiveToolbar 缺 mobile Sheet/menu 行為                                                 | Feedback/Deck/Admin 工具列仍自行處理 wrap/collapse                         | 讓 ResponsiveToolbar 接收 `filters`, `primaryAction`, `secondaryActions`，手機自動分層。       |
| Component variant  | DataList 缺 mobile card/list 模式                                                           | Admin/I18n 各寫 responsive table CSS；Leaderboard 另寫卡片                 | DataList 支援 `mobile="cards"`、`cellLabel`、`rowAction`。                                     |
| Component variant  | Card detail/popover/sheet 缺統一 surface                                                    | Card、DeckEditor、CardBrowser 各算 popover/preview                         | 新增 `CardDetailSurface` + `usePopoverPosition`。                                              |
| Responsive rule    | Touch target 44px 未成為硬規則                                                              | Landing intro、Feedback toolbar、Battle pending choice 低於要求            | 在 Button/Input/SegmentedControl/Sheet footer 強制 `min-h-11`; smoke 門檻改 44px。             |
| Responsive rule    | Hover-only fallback rule 不完整                                                             | Battle/Deck/Admin/Landing 仍靠 hover 提示可互動                            | DS 規定 hover 僅裝飾；所有 preview/tooltip 必有 click/tap/focus fallback。                     |
| Responsive rule    | Low-height desktop compact mode 缺失                                                        | 1280x720/1366x768 只能勉強顯示                                             | PageShell/Board/Deck/Toolbar 加 `@media (max-height: 780px)` 或 density prop。                 |
| Responsive rule    | Mobile secondary content disclosure 缺失                                                    | Online custom room、AI opponent deck、Feedback filters 同級展開            | 新增 disclosure pattern：Accordion/Sheet/Drawer 的使用準則。                                   |
| Interaction state  | Loading/disabled/error/success 狀態未統一                                                   | Auth/Online/Feedback/Admin 各自用 p/Badge/Panel/toast                      | 新增 `Alert`, `InlineStatus`, `LoadingState`, `Button isLoading`。                             |
| Interaction state  | Selected/pressed/ring 規格未統一                                                            | DeckSelector、Board selected card、Admin tabs、Feedback sort tabs 視覺各異 | 補 `selected` token：ring width/color/background/aria-pressed rules。                          |
| Interaction state  | Focus visible 不是全局保證                                                                  | 多數 Button 有 focus，但自寫 buttons/role button 不一致                    | 建立 `focusRing` utility，禁止移除 outline 後不補 ring。                                       |
| Motion rule        | 動畫 duration/easing/reduced motion 規則不足                                                | Battle、Landing、Auth、Tutorial 各自寫 transition/animation                | 補 `--motion-fast/base/slow`, easing token；所有重要動畫支援 prefers-reduced-motion。          |
| Accessibility rule | Dialog/Sheet focus trap、restore focus、body scroll lock 缺失                               | modal 對鍵盤與讀屏不完整                                                   | 在 shared Dialog/Sheet 實作，並要求所有 modal 遷移。                                           |
| Accessibility rule | Icon-only label 規則未集中                                                                  | 多處已標 label，但沒有 component 強制                                      | IconButton props 必填 `label`; ESLint/TS type 防漏。                                           |
| Accessibility rule | role=button card 規則缺失                                                                   | Feedback card 巢狀互動造成語意與鍵盤風險                                   | DS 禁止非必要 `role=button`；使用 button/link 或 `InteractiveCard`。                           |
| Accessibility rule | Contrast validation for custom colors 缺失                                                  | Feedback tag custom color 可能在深色背景上不足                             | 自訂 tag 顏色需計算 text tone，或限制到 token swatches。                                       |
| Accessibility rule | Table-to-card label 規則分散                                                                | Admin/I18n 各自用 CSS `data-label`                                         | DataList 統一 cell label、reading order、row action。                                          |
| Layout shell       | `html, body, #root overflow:hidden` 與 scroll pages 並存                                    | 普通文檔頁需靠各頁繞過全局 overflow                                        | PageShell 成為唯一 scroll owner；普通頁使用 `variant="scroll"`，game/tool 用 workspace。       |
| Layout shell       | PageHeader mobile action collapse rule 缺失                                                 | Lobby/Admin/Feedback/NavBar header 自寫                                    | PageHeader 支援 `actions="inline                                                               | menu"`，手機自動 drawer/menu。 |
| QA rule            | Smoke 覆蓋沒有全部必測尺寸與 Landing/DeckBuilder/Battle detail                              | PASS 結果無法代表全項目                                                    | 把補充探針轉成正式 `smoke:ui-responsive`，覆蓋 12 尺寸與主要路由。                             |
